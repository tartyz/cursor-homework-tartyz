GIT HOMEWORK:

1. Сheat list

  • git config --global user.name blabla - встановдлює глобальне імя/обліковий запис
  • git config --global user.email blabal@bla - - встановдлює e-mail
  • git init - ініціалізує Git-репозиторій
  • git status - перевірка стану
  • git add . - підготувати файл для збереження в гіті
  • git commit - зберегти файл в коміті
  • git log - показує історію комітів
  • git checkout - перевіряє гілку
  • git branch - перечід з гілки на гілку, з дописом -b може створювати нову гілку
  • git push - перекладає git зміни з локального у віддалений репозиторій
  • git pull - оновлю дані (забрати зміни) з віддаленого репозиторію по поточній гілці
  • git merge branch_name - обєднує дані/вносить зміни з вказаної гілки в локальну
  • git clone repository_address- Клонувати репозиторій, що існує

2. What is the GITIGNORE FILE

Файл Git Ignore визначає, які файли та папки не слід ігнорувати у даному вихідному коді. Його можна використовувати на основі кожного шляху, щоб правила застосовувались лише до певних папок, але ви також можете створити глобальний файл GITIGNORE, який застосовується до кожного сховища Git, яке у вас є. Ви можете знайти десятки прикладів файлів GITIGNORE, які рекомендуються в різних сценаріях, на сторінці шаблонів .gitignore GitHub.

3. Advanced part

  • git revert - ця команда створює нову фіксацію, яка скасовує зміни від попереднього виконання, додає нову історію до проекту (вона не змінює існуючу історію).
  • git rebase - перебазування змін з однієї гілки до іншої, команда подібна до merge використовується скоріше для внесення локальних змін
  • git cherry-pick - застосовує зміни введені в уже існуючих комітах
  • git stash - зберігає поточний стан в у тимчасовому місці (схованці, stash) і відновлює попередній стан, каталог стає точно таким, яким був до початку редагування.
